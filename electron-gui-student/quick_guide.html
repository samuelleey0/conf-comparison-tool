📋 Generated Files Explained
exam_info.toml (per student)
Contains exam metadata including exam name, rubrics (Major_Minor with maximum points and penalty values), and option scheme. Extracted from exam_config.toml to maintain exam context for each student's grading session.
options.ini (per student)
Stores the student scheme configuration (e.g., scheme = B). Defines the specific grading scheme and marking rules applied to that student's configuration evaluation.
solutions.toml (per student)
Comprehensive reference solution file containing exam details (name, semester, unitcode, shortname), device information (device names, types such as Switch/Router), commands executed on each device, collection settings (data_source, parallel execution, timeout), and rubric definitions. Used as the baseline for marking student configurations.
hostname/ (logs directory)
Execution logs are organized by hostname within each student's folder, containing sh_run.txt, sh_ip_int_brief.txt, and other command outputs. Maintains a clear audit trail of execution history and device interactions.

Template preparation

Device Setup Tutorial

Use this page to create the template that tells the app which devices and commands should be collected or graded.

  1. Open Device Setup and choose a setup mode: build a full template manually, create a device list only, or build from a collected logs folder.
  2. Enter a Template Name. Leave it blank only when you want to load an existing template from the saved template list.
  3. If you choose Build From Logs Folder, click Choose Logs Folder and select the collected folder that contains one folder per hostname.
  4. For manual modes, add each device, enter a clear hostname, and select the commands that should be collected or graded.
  5. When building a full template manually, attach the matching raw log file for each selected command.
  6. Click the save button at the bottom. The saved template becomes the active template used by Sample Logs and Connection.

Before You Leave

  • Every device should have a clear hostname such as R1 or S1.
  • Every required command should be selected for the correct device.
  • Baseline logs should exist when the mode requires them.
  • Use Clear Selection only when restarting the setup mode choice.
Student workspace

Directory Tutorial

Use this page to create or select the folder where student logs and session data will be saved.

  1. Open Directory before running a connection. This page prepares the student workspace where collected logs will be saved.
  2. Use Create New to create one student folder. Enter classroom, tutor name, time, student ID, and optional student name.
  3. Use Use Existing when a session already exists. Choose the session, then pick the student folder you want to work on.
  4. Use Bulk Create from File to create many student folders from CSV, TXT, XLSX, or XLS.
  5. For bulk import, leave Title row?, Header row?, and Number column? on only when your file contains those parts.
  6. After the directory is created or selected, continue to Connection to collect logs for that student.

Check These

  • Classroom, tutor name, and time should match the actual session.
  • Student IDs should be correct before folders are created.
  • Bulk import switches should match the shape of the file.
  • The selected student is the one Connection will use next.
Device collection

Connection Tutorial

Use this page to run the device queue, collect command output, and monitor live execution.

  1. Open Connection after selecting a directory and saving a device template.
  2. Review the Device Queue. Click a device row to preview its commands, or click Collect this to start from a specific device.
  3. Choose Serial or SSH. For Serial, select the matching preset or choose Custom and enter the port. For SSH, enter host, port, username, and password.
  4. Click Start Auto-Execution Sequence. Follow the prompts when the app asks you to plug in or confirm the next device.
  5. Use Activity Log for workflow messages and Terminal Output for raw command output.
  6. When collection finishes, click Next Student or Start Grading, depending on the session state.
  7. Use the reset controls only after grading is complete. Reset actions erase device configuration and cannot be undone.

During Collection

  • Do not unplug the active device while commands are running.
  • Use Activity Log for app messages and Terminal Output for raw device text.
  • Stop execution only when the wrong device or wrong connection is being used.
  • Reset devices only after grading is complete.
Review and confirm

Results Tutorial

Use this page to inspect comparison output and confirm the collected data.

  1. Open Results after collection or grading has completed.
  2. Review the comparison output for the selected student and confirm that the expected device data is present.
  3. Check command evidence and warning messages before accepting the result.
  4. If something looks missing, return to Directory or Connection and verify that the correct student and template were used.
  5. Once results are confirmed, keep the saved session data for records or review.

Look For

  • Correct student and session context.
  • Expected devices and commands.
  • Warnings or missing command output.
  • Scores that align with the rubric and collected evidence.
Baseline collection

Sample Logs Tutorial

Use this page to collect baseline logs for a template that was created as a device list first.

  1. Open Sample Logs when you need to collect baseline logs for a device-list template.
  2. Choose a save directory and enter a new folder name for the collected sample logs.
  3. Load the template that contains the devices and commands you want to collect.
  4. Select the device and commands to collect. Use Select All or Clear to adjust the command list quickly.
  5. Choose Serial or SSH connection details, then start collection.
  6. Watch the logs while collection runs. Stop collection only if the device or connection is wrong.
  7. After collection, the folder should contain one folder per hostname with raw command output files inside.

Folder Result

  • One collected folder should contain the sample run.
  • Each hostname should have its own folder.
  • Each command should save as a raw output file.
  • The collected folder can be used in Device Setup's logs-folder workflow.
