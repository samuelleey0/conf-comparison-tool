📁 Data Storage
All student data and results are stored in D:\Documents organized by exam, tutor, time slot, and student ID.
📊 Understanding the Export & Format Process

After collecting student data, the system can export and format configurations into a structured directory for comparative analysis. This process:

  • Organizes each student's configuration files into comparison_engine/students/studentID
  • Generates metadata files (exam_info.toml, options.ini, solutions.toml) for each student
  • Stores execution logs in studentID/hostname subdirectories for organization and traceability
  • Prepares data for the comparison engine to analyze configurations
  • Enables batch analysis and detailed comparison reports

📋 Generated Files Explained
exam_info.toml (per student)
Contains exam metadata including exam name, rubrics (Major_Minor with maximum points and penalty values), and option scheme. Extracted from exam_config.toml to maintain exam context for each student's grading session.
options.ini (per student)
Stores the student scheme configuration (e.g., scheme = B). Defines the specific grading scheme and marking rules applied to that student's configuration evaluation.
solutions.toml (per student)
Comprehensive reference solution file containing exam details (name, semester, unitcode, shortname), device information (device names, types such as Switch/Router), commands executed on each device, collection settings (data_source, parallel execution, timeout), and rubric definitions. Used as the baseline for marking student configurations.
hostname/ (logs directory)
Execution logs are organized by hostname within each student's folder, containing sh_run.txt, sh_ip_int_brief.txt, and other command outputs. Maintains a clear audit trail of execution history and device interactions.
⏰ When to Run Export & Format

Ideal Timing:

  • After all students complete - Once all student executions are finished and data is collected
  • Before detailed analysis - When you need comparative insights beyond individual grading
  • Before batch processing - If you plan to process multiple exams or run advanced reports
  • For archival - To create a standardized, portable format for record-keeping

Note: This step is optional for basic grading but essential for comparative analysis and detailed reporting.

💡 Helpful Tips
🔌 Connection Stability
Do not disconnect the device while execution is running. Connection interruption may result in incomplete data collection.
⚙️ Device Preparation
Always verify baseline configuration before proceeding. A proper baseline ensures accurate grading results.
✓ Data Verification
Review results immediately after completion to ensure all data was collected successfully before proceeding.
📊 Export for Analysis
Use the export & format process to prepare data for comparative analysis. This generates standardized files that the comparison engine uses for detailed reporting.
🔍 Configuration Inspection
After exporting, you can inspect individual student configurations in the comparison_engine/students directory to understand configuration variations.
📋 Additional Tools
Use System Admin for configuration and Sample Logs for validation and troubleshooting issues.
